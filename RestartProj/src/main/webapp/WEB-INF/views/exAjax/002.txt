1. http란(Hyper Text Transfet Protocol)
: 웹의 기본 프로토콜
- 동작원리 : 인터넷상 어떤 클라이언트가 서비스 요청을 할 때까지 80번 포트를 감지하고 기다린다.
			 클라이언트의 요청이 서버에 도착하면 서버는 요청을 분석하고 해당 정보를 클라이언트에게 응답한다.
			 

2. 웹페이지
: HTML문서와 script 언어가 함께 작성된 것을 말한다.

3. 스크립트 분류
- 클라이언트 스크립트 : 클라이언트 측에서 실행. ex) javascript
- 서버 스크립트 : 서버 측에서 실행. ex) Vbscript(클라이언트 스크립트로 개발되었으나 asp의 근간인 서버 스크립트로 사용되고있다.)
										asp, jsp, php
										
4. 서버 스크립트의 등장배경
- 정적 웹페이지 : 웹문서는 서버에 저장되어있는데 클라이언트의 요청이 있을 시 클라이언트로 전송된다.
				이때, 전송된 웹페이지의 내용과 서버에 저장되어있던 웹페이지의 내용이 동일하면 정적 웹페이지라고한다.
- 동적 웹페이지 : 클라이언트에서 입력한 데이터에 따라 서로 다른 응답을 전송하는 것이다.

==> 동적 웹페이지를 만들기 위해 CGI와 같은 서버 기술들이 사용 되었으나 현재는 asp, php를 많이 사용한다.


5. Html의 소개
- 웹문서를 표현하는 기술.
- <>안에 정의된 용어로 태그를 표현하므로 mark-up 언어라고 부른다.
- 대소문자 구분없음. 


6. 클라이언트 기술들
- Css(Cascading Style Sheet) : 운영체제나 웹브라우저에 관계없이 웹페이지의 레이아웃을 효과적으로 처리하는 언어.
- DHTML(Dynamic HTML) : HTML과 Css, Javascript 등 모두 동적인 웹문서를 제작할 수 있는 DHTML이라 부른다.
- Javascript : 객체 기반의 스크립트 프로그래밍 언어이다.

 
4. Web 2.0
: 단순한 웹페이지가 아닌 웹애플리케이션과 같은 웹페이지만으로 하나의 프로그램 기능을 하는 Web을 말한다.
